# Next.js Chatbot Project

This is a Next.js-based chatbot application template that uses App Router, supports Server-Side Rendering (SSR) and standalone deployment builds, suitable for deployment scenarios with native Node.js 20 runtime function services.

## Project Structure

Project structure overview:

```
├── src/                      # Application source code root directory
│   ├── app/                  # App Router pages and routes
│   ├── components/           # Reusable UI components
│   ├── lib/                  # Utility methods and common logic
│   └── middleware.ts         # Middleware
├── public/                   # Static assets directory (images, icons, etc.)
├── next.config.ts            # Next.js configuration (output: 'standalone', runtime: 'nodejs')
├── build.sh                  # Build script (generates .next/standalone)
├── run.sh                    # Startup script (node .next/standalone/server.js)
├── package.json              # Project configuration and npm scripts
├── tsconfig.json             # TypeScript configuration
├── eslint.config.mjs         # ESLint configuration
├── postcss.config.mjs        # PostCSS configuration
├── components.json           # Component/style-related configuration (e.g., UI system)
└── README.md                 # Project documentation
```

### Directory Description
- **src/app/** - Build pages and routes using Next.js App Router
- **src/components/** - Common UI components
- **src/lib/** - Utility functions and business logic encapsulation
- **src/middleware.ts** - Middleware definition (e.g., authentication, request handling)
- **public/** - Static public resources
- **next.config.ts** - Next.js core configuration (output: 'standalone', runtime: 'nodejs')
- **.next/standalone/** - Standalone build output (generated by build.sh)
- **run.sh** - Entry script, executes node .next/standalone/server.js
- **package.json** - Dependency and script management

## Development, Build, and Deployment

### 1. Development Phase

Normal development workflow:
```bash
# Install dependencies
npm install

# Start development server
npm run dev
```

### 2. Build Phase

After development is complete, build the production version:
```bash
# Run build script to compile the application
./build.sh
```

### 3. Local Testing

After the build is complete, test locally:
```bash
./run.sh
```

### 4. Deploy to Function Service

Simply upload this code package to the FaaS platform

See [Next.js Deployment Documentation](https://nextjs.org/docs/app/building-your-application/deploying) for more details.
